# Accessibility Standards

## WCAG 2.2

**WCAG 2.2 Understanding**  
https://www.w3.org/WAI/WCAG22/Understanding/

Detailed guidance for understanding and implementing WCAG 2.2 success criteria. Use this as the primary reference for accessibility requirements.

**WCAG 2.2 Quick Reference**  
https://www.w3.org/WAI/WCAG22/quickref/

Filterable, customizable reference to WCAG 2.2 success criteria. Useful for quickly finding specific requirements.

## WAI-ARIA

**WAI-ARIA 1.2 Specification**  
https://www.w3.org/TR/wai-aria-1.2/

Complete specification for ARIA roles, states, and properties. Reference this when writing ARIA attributes.

**ARIA Authoring Practices Guide (APG)**  
https://www.w3.org/WAI/ARIA/apg/

Design patterns and examples for accessible web components. Contains keyboard interaction patterns and ARIA usage.

## Color Contrast

**WebAIM Contrast Checker**  
https://webaim.org/resources/contrastchecker/

Tool to verify color contrast meets WCAG requirements. Use for testing text and background color combinations.

## Validation

**W3C Markup Validator**  
https://validator.w3.org/docs/users.html

Validates HTML markup for errors and conformance. Use to ensure semantic HTML is properly structured.
